What is MessagingTool?
MessagingToolis a tool to exchange messages with workstations including PCs. No server is needed to exchange messages; It sends a message directly to other workstations where MessagingTools are running. With MessagingTool, you can:
  • Send a message to multiple recipients.
  • Broadcast a message to a sub-network.
  • Leave a recorded message when you are out of office so that others can retrieve the recorded message.
  • Easily know whether your friends are running the tool. In addition, you can know whether they are in office or not.
  • Create a virtual meeting room.
  • Exchange a file

How To Run
  • AWT version

    • You need JRE 1.4.2 (Java Runtime Environment) from JavaSoft. If you don't have it, then please download JRE and install it. On Unix and Windows95/NT(from MS-DOS window), invoke the Java Virtual Machine like this:
    On Unix:
    % cd <directory where msgtool.jar is stored>
    % java  -jar msgtool.jar  -awt&
    On Windows (From MS-DOS window):
      >  cd <directory where msgtool.jar is stored>
      >  javaw -jar msgtool.jar  -awt
  • Swing version

    • To run a Swing version,  you need JRE 1.4.2 (Java Runtime Environment) from JavaSoft. If you don't have it, then please download JRE and install it. On Unix and Windows95/NT(from MS-DOS window), invoke the Java Virtual Machine like this:
    On Unix:
    % cd <directory where msgtool.jar and swingall.jar are stored>
    % java  -jar  msgtool.jar  &
    On Windows (From MS-DOS window):
      >  cd <directory where msgtool.jar and swingall.jar are  stored>
      >  javaw -jar  msgtool.jar

Message Sender(1.00)
Message Sender included in the msgtool.jar file is a command line tool  to send a message to the version 2.00 or later versions  of MessagingTool.   Its command line syntax is following:
java -cp msgtool.jar MsgSend <Recipient> <Message>
where <Recipient> is a name of recipient or double-quoted list of recipients separated by comma. For example:
java -cp msgtool.jar MsgSend Yoshiki"How are you?"
or
java -cp msgtool.jar MsgSend "Yoshiki, Taro" "How are you?"
The Address Cache file is shared by both MessagingTool and Message Sender. To edit the Address Cache file, you have to use the edit menu of MessagingTool. Other files such as Properties and Recipient Hints are also shared. The Recipient Hints is used to expand elements in <Recipient> into members defined in the Recipient Hints. For example, suppose that there is an entry in the Recipient Hints which looks like this:
Recipient: MyGroup
Members: David, Smith, Lewis
Then if you specify MyGroup as <Recipient> like below, the  message is sent to David, Smith, and Lewis:
java -cp msgtool.jar MsgSend MyGroup "Good Morning! How are you?"
