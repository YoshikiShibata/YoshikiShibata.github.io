MessagingTool Release Notes

README FIRST:
The MessagingTool is NOT COMPATIBLE with JDK1.1, JDK1.2, and JDK1.3. You must use Java 2 SDK1.4 or later versions..

To specify a directory where all files such as property, address caches, etc are stored, please use -Duser.home option like this:

  • Windows

    • javaw -Duser.home=XXXXXX -jar msgtool.jar
  • Linux/Solaris

    • java  -Duser.home=XXXXXX -jar msgtool.jar
where XXXXXX is a full path name of the directory.

The behavior of passing a "xxx.xxx.xxx.xxx" style address string(raw IP address string such as 129.249.53.58) to the socket interface seems dependent on the implementation of JDK. If you have any problem with that style addressing, then try a host name instead. As far as I know, with Windows 95, you need to specify host name instead of IP address. I hope this problem will be fixed with JDK 1.1.
